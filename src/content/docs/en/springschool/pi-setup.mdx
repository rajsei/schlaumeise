## Starting Point

The Raspberry Pi OS has already been installed on the SD card.

Typically, this is done using the Raspberry Pi Imager, which allows configuration of settings such as Wi-Fi credentials, hostname, and passwords during the setup process.

In our case, these steps have already been completed for you, so the system is ready to use.


## Instructions

<Steps>
  1. **Insert the SD card into the Raspberry Pi** and **connect the Raspberry Pi to the power supply** using the power adapter.
   An LED on the Raspberry Pi should light up or flash. 
   It is best to wait a few minutes for it to boot up; the initial boot process takes longer. 
   Do not disconnect it from the power supply during this time.

  2. Ensure that the computer is connected to the Wi-Fi network. *Wifi:* `birdnet-wifi`, *Password:* `rotkehlchen`.

  3. Open the console.
</Steps>

      If the following message appears:
      <Code

      this means that no connection can be established to your Raspberry Pi.

      Check the following points:
      <Steps>
        1. Power supply: Make sure that the Raspberry Pi is turned on, the SD card is inserted correctly,
        and the status LEDs are flashing or lit.
        2. Network: Make sure that your PC and Raspberry Pi are on the same network—i.e., connected to the same Wi-Fi or hotspot.
        (Make sure that the Wi-Fi data in the Raspberry Pi OS matches your current Wi-Fi.)
        3. Reinstallation: If the connection still fails, reinstall Raspberry Pi OS on the SD card
        and carefully re-enter the Wi-Fi access data.
        4. Direct connection: Experienced users can connect a keyboard and screen directly to the Raspberry Pi
        to manually check the connection and network settings.
      </Steps>
    </AccordionItem>
    <AccordionItem title="What if ‘WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!’ is displayed?">

      Don't worry—this is normal if you have previously connected to a
      Raspberry Pi with the same name via SSH. Here's how to fix the problem:

      <Steps>
        1. Open the file: C:\Users\username\\.ssh\known_hosts (the exact path is also in the warning) with a text editor.

        2. Find all lines that start with birdnet-rpi00.local and delete them.
        There may be several entries spanning multiple lines.

        3. If you have never used SSH for other connections, you can also delete the entire contents of the file. 4. Save the file, close the editor, and then try the connection again.

     </Steps>

<div class="step-code">
Due to its limited RAM (512 MB) of the Raspberry Pi Zero 2 W, the installation of BirdNET-Pi will fail without increasing the swap size.
  Increasing swap allows part of the SD card to be used as additional virtual memory.
  In addition, the Wi-Fi power options must be adjusted for later use of the BirdNET-Pi.
```sh
sudo sed -i 's/CONF_SWAPSIZE=512/CONF_SWAPSIZE=2048/g' /etc/dphys-swapfile &&
sudo sed -i 's/#CONF_MAXSWAP=2048/CONF_MAXSWAP=4096/g' /etc/dphys-swapfile
```
