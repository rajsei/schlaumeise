## When should I do this?

If your BirdNET Pi is only set up inside your Wi-Fi network, you can skip the following steps.
In this section we will make settings so that if your BirdNET Pi is outside your Wi-Fi network, it works as a hotspot and emits its own Wi-Fi network (this has no internet access), so you can connect from your phone to the BirdNET Pi via Wi-Fi and access the data of the BirdNET Pi.
You can also e.g. add other Wi-Fi connections here or change Wi-Fi data already entered in Raspberry Pi OS setup.

## Instructions

<Steps>

0. Connect via ssh.
1. Now we edit the Wi-Fi and hotspot settings. For this we first have to open the NetworkManager:

</Steps>
<div class="step-code">

```sh
sudo nmtui
```

![](/schlaumeise//images/project01/pi-setup2/nmtui01.jpg)

</div>
<Steps>

2. In NetworkManager you can move the selected item with the <Kbd windows="Arrow keys" /> and confirm with <Kbd windows="Enter" />.

3. Select `Edit a connection`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui01.jpg)

</div>
<Steps>

4. Select `Add`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui02.jpg)

</div>
<Steps>

5. Select `Wi-Fi` as connection type.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui03.jpg)

</div>
<Steps>

6. Select `Create`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui04.jpg)

</div>
<Steps>

7. Now you have to add the data of your network.
</Steps>
<div class="step-code">
- `Profile Name` is only for identification, you can enter any name here. But make sure to replace Hotspot with your chosen name in the following commands.
- `Device` is wlan0
- `SSID` is the name of the Wi-Fi network created, you can also choose this freely, but remember it!


![](/schlaumeise//images/project01/pi-setup2/nmtui05.jpg)

</div>
<Steps>

8. Select `Access Point` for `Mode`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui06.jpg)

</div>
<Steps>

9. Select `B/G (2.4 GHz)` for `Channel`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui08.jpg)

</div>
<Steps>

10. For `Security`, select `WPA & WPA2 Personal`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui09.jpg)

</div>
<Steps>

11. Enter the hotspot password under `Password`. Remember it!

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui10.jpg)

</div>


<Steps>

12. Configure the **IP settings** as follows:

</Steps>
<div class="step-code">
- IPv4 Configuration: `Shared`
- IPv6 Configuration: `Disabled`

![](/schlaumeise//images/project01/pi-setup2/nmtui20.png)

</div>


<Steps>

13. Now we can close the program. Go down to `OK`.
14. Go to `Back`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui11.jpg)

</div>
<Steps>

15. Go to `Quit`.

</Steps>
<div class="step-code">

![](/schlaumeise//images/project01/pi-setup2/nmtui12.jpg)

</div>
<Steps>

16. Now set the connection priorities for Wi-Fi and hotspot connection.
    This way the Raspberry Pi will try to connect to the known Wi-Fi network first.
    Only if no connection is possible, it will automatically create its own hotspot so that a connection to the device can still be made.
    First we need to navigate to the correct folder with the `cd` command.

</Steps>
<div class="step-code">

```
cd /etc/NetworkManager/system-connections
```

18. Add the following code snippet in the first part of the file.
    The entry `autoconnect=true` activates automatic connection.
    The value `autoconnect-priority=0` sets the priority of the network.
    The higher the number, the more preferred that network is during auto-connect.

</Steps>

```

autoconnect-priority=0
```
