## Voraussetzungen

Du benötigst einen Computer, eine passende SD-Karte für deinen Raspberry Pi, auf der das richtige Betriebssystem installiert ist
(siehe „SD-Karte einrichten”), sowie eine Internetverbindung. …

## BirdNET Pi installieren

<Steps>

1. Stecke die SD-Karte in den Raspberry Pi.

2. Schließe den Raspberry Pi mithilfe des Netzteils an den Strom an.
   Es sollte eine LED am Raspberry Pi leuchten oder blinken. Warte am besten ein paar Minuten, damit er hochfahren kann.

3. Öffne die Konsole.

</Steps>

<Aside type="caution" title="Fehlerbehebung"> 
  <Accordion>
    <AccordionItem title="Was wenn keine Verbindung hergestellt werden kann? (Der angegebene Host ist unbekannt.)">

      Wenn die Meldung erscheint:
      <Code
      lang="plaintext" title="Terminal" class="terminal-code"
      code="PS C:\Users\Name> ssh birder@birdnetpi.local
      ssh: Could not resolve hostname birdnetpi.local: Der angegebene Host ist unbekannt."
      />
      bedeutet das, dass keine Verbindung zu deinem Raspberry Pi hergestellt werden kann.

      Überprüfe folgende Punkte:
      <Steps>
        1. Stromversorgung: Stelle sicher, dass der Raspberry Pi eingeschaltet ist, die SD-Karte korrekt eingesteckt ist
        und die Status-LEDs blinken oder leuchten.
        2. Netzwerk: Vergewissere dich, dass PC und Raspberry Pi im selben Netzwerk sind – also mit demselben WLAN oder Hotspot verbunden.
        (Achte darauf, dass die WLAN-Daten im Raspberry Pi OS mit deinem aktuellen WLAN übereinstimmen.)
        3. Neuinstallation: Wenn die Verbindung weiterhin fehlschlägt, installiere Raspberry Pi OS neu auf der SD-Karte
        und trage die WLAN-Zugangsdaten erneut sorgfältig ein.
        4. Direkte Verbindung: Erfahrene Nutzer können eine Tastatur und einen Bildschirm direkt am Raspberry Pi anschließen,
        um die Verbindung und Netzwerkeinstellungen manuell zu überprüfen.
      </Steps>
    </AccordionItem>
    <AccordionItem title="Was wenn 'WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!' angezeigt wir? ">

      Keine Sorge – das ist normal, wenn du bereits früher per SSH mit einem
      Raspberry Pi gleichen Namens verbunden warst. So behebst du das Problem:

      <Steps>
        1. Öffne die Datei: C:\Users\Benutzername\.ssh\known_hosts (der genaue Pfad steht auch in der Warnung) mit einem Texteditor.
        2. Suche alle Zeilen, die mit birdnetpi.local beginnen, und lösche sie.
        Es können mehrere Einträge sein, die sich über mehrere Zeilen erstrecken.
        3. Wenn du SSH noch nie für andere Verbindungen genutzt hast, kannst du auch den gesamten Inhalt der Datei löschen.
        4. Speichere die Datei, schließe den Editor und versuche die Verbindung anschließend erneut.
      </Steps>

    </AccordionItem>

<Steps>

7. Dieser Schritt ist nur für Raspberry Pi Zero 2 W:
   Der Arbeitsspeicher des Raspberry Pi Zero 2 W ist zu gering, um BirdNET-Pi ohne Anpassung zu installieren.
   Daher muss die sogenannte Swapsize erhöht werden. Sie sorgt dafür, dass zusätzlich zur physischen Hardware
   auch ein Teil der SD-Karte als Arbeitsspeicher genutzt wird.
   Ohne diese Anpassung lässt sich BirdNET-Pi auf dem Zero 2 W nicht installieren.
   Zudem müssen die WLAN-Energieoptionen angepasst werden.

</Steps>
<div class="step-code">

```sh
sudo sed -i 's/CONF_SWAPSIZE=512/CONF_SWAPSIZE=2048/g' /etc/dphys-swapfile &&
sudo sed -i 's/#CONF_MAXSWAP=2048/CONF_MAXSWAP=4096/g' /etc/dphys-swapfile &&
sudo sed -i '/^exit 0/i sudo iw wlan0 set power_save off' /etc/rc.local
```

<Aside type="caution" title="Änderungen prüfen">


Starte den Raspberry Pi jetzt neu.

```sh
sudo reboot
```

Warte anschließend einige Minuten, bis das System vollständig gestartet ist und sich mit dem WLAN verbunden hat.
Verbinde dich danach erneut per SSH mit dem Pi (siehe Schritt 4 und 6).

<Steps>

8. Mit diesem Befehl startest du die Installation von BirdNET-Pi.
   Der Vorgang kann mehrere Minuten dauern – je nach Internetverbindung und Systemleistung.
   Lass das Terminal geöffnet, bis die Installation vollständig abgeschlossen ist.

</Steps>
